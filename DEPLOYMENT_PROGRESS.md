### ⭐ أبرز القرارات التقنية

**1. Multi-Stage Build للـ Backend:**
- Stage 1 (`composer-deps`): يثبّت vendor فقط — **مكش يُعاد بناؤه** عند تغيير الكود
- Stage 2 (`runtime`): PHP-FPM مع كل الـ extensions
- النتيجة: Docker cache فعّال + صور أصغر بكثير

**2. PHP Extensions المثبّتة:**
```
pdo_mysql  → قاعدة البيانات
redis      → phpredis (أسرع من predis)
gd         → معالجة الصور
zip        → ملفات مضغوطة
bcmath     → عمليات حسابية دقيقة
intl       → اللغات والتواريخ
pcntl      → Queue Worker signals
gmp        → VAPID لـ Web Push Notifications
opcache    → JIT compiler (30-50% performance boost)
```

**3. Entrypoint Script الذكي:**
- ينتظر MySQL يكون جاهزاً قبل التشغيل (max 60 ثانية)
- ينتظر Redis يكون جاهزاً
- يشغّل `php artisan migrate --force` تلقائياً
- يعمل `config:cache` و`route:cache` في الإنتاج
- يُنشئ `storage:link`

**4. Queue Workers ×2 Replicas:**
- مشروع وداد عنده 9 Jobs — Queue Worker واحد قد لا يكفي
- `ProcessChatbotMessageJob` و`ProcessLabTestJob` ممكن يكونوا long-running
- Worker ثاني يضمن معالجة Jobs بالتوازي

**5. Scheduler بدون Cron Daemon:**
```bash
# بدلاً من crontab، container مستقل بـ while loop:
while true; do php artisan schedule:run; sleep 60; done
```
- أنظف من cron في Docker
- السجلات تظهر في `docker compose logs scheduler`

**6. Nginx Timeout 120 ثانية:**
```nginx
fastcgi_read_timeout 120s;  # للـ Chatbot والـ OCR التي تستغرق وقتاً
```
- الـ Default هو 60 ثانية — غير كافي لطلبات AI

**7. MySQL UTF8MB4:**
- يدعم العربية والـ Emoji بالكامل
- `utf8mb4_unicode_ci` — الـ collation الصح للعربية

**8. Redis persistence (AOF):**
```
--appendonly yes --appendfsync everysec
```
- يحفظ البيانات على disk — مش بتتضيع لو الـ container وقف
